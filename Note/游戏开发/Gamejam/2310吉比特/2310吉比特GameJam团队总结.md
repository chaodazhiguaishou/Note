
# 策划
1. 一开始的需求还是没定好就着急开始了，之后还是得在 idea 确定之后把所有需求能想好的都想好，并且需求中必须做到的点也要沟通好写下来。
2. 设计引导的时候要站在玩家的角度，不能先入为主。在沟通绘制引导漫画的时候，应该定好哪部分是需要保留的哪部分是可以随意发挥的。
3. 关卡设计和数值设计最好还是提前准备好，不要等到 demo 出了再安排关卡设计和数值（前面关卡的球如何来，第六关两个基地是不是也要相应修改球的波次速度等）
4. 提需求的时候可以不只写表，可以直接把游戏草图都画出来在里面标明需求。（比如直接把游戏里的 UI 排布画好并且说好哪些按钮有哪些功能）

# 程序
1. 游戏 demo 重启后关卡文件清空了，应该还是得保留进度的
2. GameJam 尽量还是不要搞创新，可以复用的就复用保证效率。不过这次很多复用还是有些失误的，之后可以尽量每次搞完一个项目都技术总结一下，把实现的东西拆开成组件放在哪个地方以备不时只需。
3. 不要忘记音乐和音效，提前留好接口或者先随便用个音乐代替。不要最后时刻再加音乐。
4. 代码规范问题，一定要留好接口和注释。并且要保证先测试找 bug，像这次的不匀速以及对象池复用出来未重置的问题应该尽早排查好。

# 美工
1. 以后尽量美工也加入 unity 使用，并且最好也可以学会使用下分支的 pullpush，这样每次 demo 搭建的时候美工直接可以看效果。（比如教程的背景和字的颜色的搭配，游戏内的猫猫与身后的球效果）
2. 美工在 unity 理安排好自己的文件结构后就不要再移动文件，不然可能导致 demo 材质丢失。后续尽量保证加素材而不删素材，所以尽量命名的时候要说清楚是初版、优化版、或者补的什么，并且有新的素材要及时检查有没有修改。
3. 素材格式问题，如果是多个素材在同一张图中，要保证该张图可以切割成 m * n 个矩形（最好是正方形）的规格，每个素材都在一个矩形中。然后如果是动画素材，要保证其在每个矩形的同一位置。
# 音效
1. 非常好音效，爱来自 AI。用 ai 作曲网站 Soundraw 搞背景音乐真的是神来之笔。
2. 音乐声音还是要控制一下，在 demo 整合的时候要注意不要太大声。


# 总结

策划：需求表最好结合草图一起给出来；需求表要在第二天结束前就核对好，第三天专心搭建demo
程序：gj 的时候注重模块复用；打完比赛及时技术总结。
美工：简单学习一下 unity 和版本控制可以更好监督下其他人的进度；和一些素材格式
